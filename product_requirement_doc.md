# PRD：产品标准合规推荐助手

## 1. 产品名称

产品标准合规推荐助手  
Product Standard Compliance Recommendation Agent

## 2. 产品定位

面向商品选购、采购辅助和产品合规初筛的 AI Workflow 工具。它通过标准检索、证据判断和推荐排序，帮助用户判断候选商品是否具备足够公开证据。

## 3. 目标用户

| 用户 | 需求 |
|---|---|
| 普通消费者 | 购买儿童用品、食品接触用品、家电等商品前做安全初筛 |
| 企业采购 | 快速排除标准信息不透明的商品 |
| 内容测评团队 | 在测评前整理标准和证据线索 |
| AI 产品经理 | 展示 Agent 流程设计和 Prompt 风控能力 |

## 4. 使用场景

- 家长购买儿童保温杯、儿童餐具、学习用品。
- 企业采购办公设备、清洁用品、劳保用品。
- 内容团队做商品合规信息整理。
- 产品经理作品集展示 AI Workflow 项目。

## 5. 用户痛点

| 痛点 | 说明 |
|---|---|
| 标准难查 | 用户不知道某类产品适用哪些标准 |
| 证据难判 | 商品页面信息分散，难以判断可信度 |
| 推荐不透明 | 普通推荐缺少标准依据和风险提示 |
| 幻觉风险 | LLM 可能编造标准、链接或检测报告 |

## 6. 用户故事

作为一名家长，我希望输入“儿童保温杯，预算 100 元以内，要求安全、防漏、材质合规”，系统能告诉我应该关注哪些标准，并推荐有公开证据的商品，这样我可以减少踩坑风险。

作为一名采购人员，我希望快速筛掉缺少执行标准和检测信息的商品，这样可以提升采购初筛效率。

## 7. 功能列表

| 功能 | 优先级 | 说明 |
|---|---|---|
| Webhook 输入 | P0 | 接收结构化产品需求 |
| 输入校验 | P0 | 缺少 product_name 时返回错误 |
| 品类识别 | P0 | 识别产品类别和合规关注点 |
| 标准搜索 | P0 | 查找标准、认证和监管信息 |
| 标准整理 | P0 | 提取标准编号、名称、适用范围 |
| 商品搜索 | P0 | 查找商品页面和公开证据 |
| 合规分析 | P0 | 判断证据等级和不确定性 |
| 推荐排序 | P0 | 使用 100 分制输出推荐等级 |
| 兜底报告 | P1 | 搜索结果不足时返回风险说明 |
| Mock Demo | P1 | 无搜索 API 时演示流程结构 |

## 8. 输入字段

| 字段 | 类型 | 必填 | 示例 |
|---|---|---|---|
| product_name | string | 是 | 儿童保温杯 |
| use_case | string | 否 | 小学生日常上学使用 |
| budget | string | 否 | 100元以内 |
| region | string | 否 | 中国大陆 |
| extra_requirements | string | 否 | 安全、防漏、材质合规 |

## 9. 输出字段

| 字段 | 说明 |
|---|---|
| category_judgement | 产品类别判断 |
| standards_table | 相关标准表格 |
| selection_criteria | 选品判断标准 |
| recommended_products | 推荐商品列表 |
| evidence_level | 证据等级 |
| total_score | 综合评分 |
| uncertainty | 不确定性说明 |
| risk_warnings | 风险提醒 |
| source_links | 信息来源 |
| disclaimer | 免责声明 |

## 10. 核心流程

1. 接收用户输入。
2. 校验 `product_name`。
3. 识别产品品类。
4. 搜索标准信息。
5. 整理标准和证据来源。
6. 生成商品搜索策略。
7. 搜索商品公开信息。
8. 分析商品证据等级。
9. 评分排序。
10. 输出最终报告。

## 11. 异常流程

| 异常 | 处理方式 |
|---|---|
| 缺少 product_name | 返回错误 JSON，不继续执行 |
| 标准搜索无结果 | 输出“未找到足够的公开标准信息”兜底报告 |
| 商品搜索无结果 | 输出“不输出强推荐商品”兜底报告 |
| API 调用失败 | 返回调试建议和手动检查点 |
| 商品证据不足 | 降级推荐或不推荐 |

## 12. 风险控制

- 不允许编造标准编号。
- 不允许编造商品链接。
- 不允许编造检测报告。
- 不能把可能符合写成确定符合。
- 每个商品必须输出证据等级和不确定性。
- 涉及儿童、食品接触、电器等品类时证据不足必须降级。

## 13. 成功指标

| 指标 | 说明 |
|---|---|
| 标准召回率 | 能否找到与产品品类相关的标准线索 |
| 证据完整度 | 推荐商品是否有可追溯来源 |
| 幻觉控制率 | 是否避免编造标准、链接和检测报告 |
| 报告可读性 | 普通用户是否能理解推荐原因 |
| 推荐可解释性 | 每个推荐是否有证据和风险说明 |

## 14. 当前版本边界

当前版本是可运行 Demo，不是生产级合规系统。它适合做公开信息初筛和作品集展示，不替代检测机构、监管结论或法律意见。
