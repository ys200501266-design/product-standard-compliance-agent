# PRD：产品标准合规推荐助手

## 1. 产品名称

产品标准合规推荐助手  
Product Standard Compliance Recommendation Agent

## 2. 产品目标

帮助用户在购买商品前快速了解相关生产执行标准、国家标准、行业标准或认证要求，并基于公开证据筛选和推荐更可信的商品。

## 3. 目标用户

- 对商品安全和标准合规有要求的普通消费者。
- 需要做采购初筛的运营、行政、供应链或采购人员。
- 需要展示 AI 工作流能力的产品经理、AI 产品经理或自动化从业者。
- 对消费品标准、认证和检测报告敏感的内容或导购团队。

## 4. 使用场景

- 家长购买儿童水杯、保温杯、餐具等食品接触类用品。
- 企业采购办公电器、劳保用品、清洁用品。
- 内容团队做商品测评前的标准资料整理。
- AI 产品经理作品集展示工作流设计能力。

## 5. 用户故事

作为一名家长，我想输入“儿童保温杯，预算 100 元以内，要求安全、防漏、材质合规”，系统能告诉我应该关注哪些标准，并推荐有公开证据的商品，这样我可以减少踩坑风险。

作为一名采购人员，我想快速知道某类商品的关键标准和候选产品证据链，这样我可以把不透明或证据不足的商品排除在初筛之外。

## 6. 功能列表

| 功能 | 说明 |
|---|---|
| 用户需求接收 | 通过 Webhook 接收结构化输入 |
| 品类识别 | 判断产品类别、子类别和风险点 |
| 标准检索 | 查找国家标准、行业标准、认证和检测信息 |
| 标准整理 | 提取编号、名称、适用范围、关键要求和来源 |
| 商品检索 | 根据标准关键词查找商品和证据页面 |
| 合规分析 | 判断商品是否公开标注标准或认证 |
| 推荐排序 | 按权重评分并输出推荐 |
| 报告生成 | 返回 Markdown 结构化报告 |

## 7. 输入字段

| 字段 | 必填 | 示例 |
|---|---|---|
| product_name | 是 | 儿童保温杯 |
| use_case | 否 | 小学生日常上学使用 |
| budget | 否 | 100元以内 |
| region | 否 | 中国大陆 |
| extra_requirements | 否 | 安全、防漏、材质合规、有执行标准说明 |

## 8. 输出字段

- 产品类别判断
- 相关标准表格
- 标准关键要求总结
- 选品判断标准
- 推荐产品表格
- 最推荐产品
- 性价比最高产品
- 风险提醒
- 信息来源链接
- 免责声明

## 9. 业务流程

1. 用户提交产品需求。
2. 系统识别产品品类和合规关注点。
3. 系统检索标准和认证信息。
4. 系统整理标准信息。
5. 系统生成商品搜索策略。
6. 系统检索商品和公开证据。
7. 系统判断每个商品的证据状态。
8. 系统评分排序。
9. 系统生成最终报告。

## 10. 异常情况

| 异常 | 处理方式 |
|---|---|
| 未识别产品品类 | 输出候选品类和需要补充的信息 |
| 未找到官方标准 | 明确写“暂无公开证据证明”，并列出已搜索来源 |
| 未找到商品链接 | 不编造链接，提示需要更换搜索 API 或补充平台 |
| 价格缺失 | 标注“未找到公开价格” |
| 标准适用性不确定 | 标注“需结合产品材质和用途确认” |

## 11. 风险控制

- 所有结论必须基于搜索结果或输入数据。
- 对未找到证据的内容不能推断为符合。
- 标准编号、商品价格和链接不得由模型编造。
- 输出免责声明，不构成检测结论或法律意见。
- 推荐结果必须保留证据链接和证据可信度。

## 12. 成功指标

- 用户能在 1 分钟内获得结构化选品报告。
- 报告中每条标准和商品推荐都有来源或证据状态。
- 模型输出中无虚构标准编号和虚构商品链接。
- 普通用户能理解推荐理由和风险提醒。
- 作品集展示时能清楚体现工作流和 Prompt 设计能力。
