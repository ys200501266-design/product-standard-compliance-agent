# GitHub 作品集展示建议

## 1. 仓库简介怎么写

建议填写：

> Evidence-based AI workflow for product standard compliance screening and recommendation.

中文作品集可以写：

> 一个基于 n8n 和 LLM 的商品标准合规推荐工作流，先查标准，再看证据，最后生成推荐报告。

## 2. GitHub Topics

建议添加：

- ai-product-manager
- n8n
- llm-agent
- workflow-automation
- prompt-engineering
- product-compliance
- recommendation-system
- evidence-based-ai

## 3. README 第一屏怎么优化

第一屏需要让人马上看懂三个问题：

1. 这是做什么的。
2. 为什么不是普通商品推荐。
3. 能展示哪些 AI 产品经理能力。

建议第一屏放：项目标题、一句话介绍、Project Status、能力标签和核心流程图。

## 5. 如何在简历里写

示例：

> 设计并搭建“产品标准合规推荐助手”AI Workflow，使用 n8n 串联 LLM、搜索 API、证据分级和推荐排序，实现从用户需求到标准检索、商品证据判断和结构化报告生成的自动化流程。项目重点解决 LLM 幻觉、推荐可解释性和合规初筛风险控制问题。

## 6. 如何在作品集网站里写

可以按以下结构展示：

1. 背景和问题。
2. 用户痛点。
3. 工作流设计。
4. Prompt 风控。
5. 证据等级模型。
6. 评分规则。
7. Demo 输出。
8. 局限和后续计划。

## 7. 面试时怎么讲

面试时不要只说“我做了一个 n8n 工作流”。更好的讲法是：

> 我把一个容易产生幻觉的商品推荐问题，改造成了证据驱动的合规初筛流程。系统先查标准，再看商品证据，最后按可解释规则推荐。

## 8. 哪些地方不要夸大

- 不要说它能证明商品一定合规。
- 不要说它能替代检测机构。
- 不要说示例商品是真实推荐。
- 不要说搜索结果一定完整。
- 不要隐藏当前版本依赖搜索 API 的事实。
