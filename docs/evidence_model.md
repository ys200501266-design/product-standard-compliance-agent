# Evidence Model：证据等级模型

这个模型用于判断商品合规证据的可信度。它不是法律意义上的认证结论，而是帮助工作流区分“官方事实”“品牌声明”“电商线索”“第三方线索”和“无证据”的产品化规则。

## 证据等级表

| 证据等级 | 来源类型 | 可信度 | 示例 | 使用方式 |
|---|---|---:|---|---|
| A | 政府/国家标准平台/监管机构 | 最高 | 国家标准全文公开系统、全国标准信息公共服务平台、市场监管总局 | 用于确认标准存在、现行状态和适用范围 |
| B | 品牌官网/官方检测报告/官方旗舰店 | 高 | 品牌官网参数页、检测报告 PDF、官方旗舰店详情页 | 用于判断商品是否公开标注标准或认证 |
| C | 电商详情页 | 中 | 京东/天猫商品详情页、商品参数区、详情图 | 可作为商品标注线索，但需要二次确认 |
| D | 第三方文章/论坛/问答 | 低 | 导购文章、测评帖、问答平台 | 只能作为补充线索，不能单独支撑强推荐 |
| E | 无公开证据 | 不可信 | 找不到标准、认证、检测或包装标识说明 | 不允许强推荐 |

## 证据等级如何影响推荐

| 证据等级 | 推荐影响 |
|---|---|
| A | 可用于确认标准本身，但仍需商品层面的 B/C 证据证明商品标注了该标准 |
| B | 商品层面强证据，推荐排序中优先级最高 |
| C | 可作为候选商品线索，但需要在风险提醒中说明仍需购买前确认 |
| D | 只能辅助搜索，不应直接作为推荐依据 |
| E | 不允许强推荐，最多输出“不建议基于当前信息购买” |

## 证据分建议

| 证据等级 | 证据可信度得分 |
|---|---:|
| A | 23-25 |
| B | 21-25 |
| C | 15-20 |
| D | 5-14 |
| E | 0-4 |

说明：A 级通常用于标准确认，B/C 级用于商品确认。一个商品即使关联了 A 级标准，如果商品页面没有标注，也不能直接判定为符合。

## 什么时候必须输出“暂无公开证据证明”

出现以下任一情况，必须输出“暂无公开证据证明”或更完整的风险表述：

1. 搜索结果没有出现明确标准编号。
2. 商品页面没有出现执行标准、认证、检测报告或包装标识说明。
3. 只有“食品级”“母婴级”“安全材质”等营销词。
4. 来源是第三方文章，但没有链接到品牌官网、检测报告或官方平台。
5. 模型只能根据常识推断，无法从输入或搜索结果中找到证据。

推荐使用完整句式：

> 暂无公开证据证明该产品完全符合该标准。

## 为什么电商宣传语不能等同于合规证据

电商宣传语常用于营销，不一定对应具体标准编号、检测项目或适用范围。例如“食品级不锈钢”并不自动说明该产品公开标注了食品接触材料标准，也不说明检测报告是否覆盖杯体、杯盖、密封圈等组件。

因此，电商页面可以作为线索，但需要继续查看参数区、详情图、包装图、检测报告或品牌官方说明。

## 为什么 LLM 不能自动补全标准编号

标准编号具有强事实属性。即使某个品类常见某些标准，模型也不能在当前任务中凭常识补全。原因包括：

- 标准可能已废止、替代或即将实施。
- 同一品类可能因材质、用途、年龄段不同适用不同标准。
- 商品页面标注的执行标准可能是企业标准或旧标准。
- 错误标准编号会直接误导购买决策。

## 如何区分事实、推断和不确定性

| 类型 | 定义 | 输出方式 |
|---|---|---|
| 事实 | 来源中明确出现的信息 | 写入 `evidence_summary`，附来源链接 |
| 推断 | 基于品类和上下文的合理判断 | 必须标注“推断”或“可能” |
| 不确定性 | 当前证据无法确认的部分 | 写入 `uncertainty` 和风险提醒 |

工作流的原则是：事实可用于推荐，推断只能辅助解释，不确定性必须保留。
