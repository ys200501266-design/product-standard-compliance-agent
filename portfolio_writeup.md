# 作品集文案：产品标准合规推荐助手

## 项目背景

消费者在购买母婴、食品接触、家电、个护等商品时，往往会看到大量营销话术，但真正影响安全和合规判断的信息，例如执行标准、检测报告、认证信息和材质说明，通常分散在标准平台、品牌官网和商品详情页中。普通用户很难判断“这个商品看起来安全”和“这个商品有公开证据证明符合相关标准”之间的区别。

我设计了一个基于 n8n 的 AI 工作流项目：产品标准合规推荐助手。它面向商品选购、采购辅助和产品合规初筛场景，通过“先查标准、再找商品、再看证据、最后推荐”的流程，帮助用户获得更可靠的选购建议。

## 用户痛点

- 不知道某类产品应该看哪些国家标准或行业标准。
- 不知道商品详情页里的标准编号是否真实、是否相关。
- 只看销量和价格容易忽视材质、安全和检测信息。
- 手动检索标准和商品证据耗时，且缺乏统一判断框架。

## 我的产品思路

我没有把它设计成普通商品推荐机器人，而是拆成三个核心问题：

1. 这个产品属于什么合规品类？
2. 这个品类应该关注哪些标准和认证？
3. 市面上的商品是否公开提供了足够证据？

因此，工作流先用 LLM 做品类识别和检索关键词生成，再调用搜索 API 查找标准来源，随后让 LLM 提取标准信息并生成商品搜索策略。商品检索完成后，系统只基于公开证据分析合规状态，最后按照评分规则输出推荐结果。

## 工作流设计

工作流包含 Webhook、6 个 LLM 节点、2 个搜索节点和 Respond to Webhook 节点：

- Webhook 接收产品名称、使用场景、预算、购买地区和特殊要求。
- LLM 1 识别品类，并生成标准检索关键词。
- Search 1 检索国家标准、行业标准、认证和检测信息。
- LLM 2 整理标准编号、名称、适用范围、关键要求和证据链接。
- LLM 3 生成商品搜索关键词组合。
- Search 2 查找商品、品牌官网、电商详情页和检测报告。
- LLM 4 判断商品是否有公开证据。
- LLM 5 根据评分规则排序。
- LLM 6 输出最终 Markdown 报告。

## Prompt 设计

Prompt 的重点不是让模型“尽量回答”，而是让模型“知道何时不能下结论”。我在每个节点里加入了禁止事项：

- 不允许编造标准编号。
- 不允许编造产品链接。
- 不能把“可能符合”写成“确定符合”。
- 没有证据时必须写“暂无公开证据证明”。
- 必须区分官方来源、品牌官网、电商页面和第三方线索。

这种设计能降低大模型在合规场景中的幻觉风险，也让输出更适合作为采购或选品的初筛依据。

## 评分规则

推荐排序使用五个维度：

| 维度 | 权重 | 含义 |
|---|---:|---|
| 标准匹配度 | 40% | 是否公开标注相关标准或认证 |
| 信息可信度 | 20% | 证据来源是否来自官方、品牌官网或明确商品详情页 |
| 价格合理性 | 15% | 是否符合预算和同类价格水平 |
| 品牌可信度 | 15% | 品牌是否有可验证官网或官方旗舰店信息 |
| 用户需求匹配度 | 10% | 是否满足场景、材质、防漏、安全等要求 |

## 输出示例

用户输入“儿童保温杯，预算 100 元以内，要求安全、防漏、材质合规、有执行标准说明”后，系统会输出：

- 产品类别判断：食品接触用不锈钢真空保温容器/儿童水杯。
- 相关标准表格：标准编号、标准名称、适用范围、关键要求和来源链接。
- 选品判断标准：必须满足、优先满足和需要避坑的风险点。
- 推荐产品表格：产品名称、品牌、参考价格、证据可信度、推荐理由和风险提醒。
- 最终结论：最推荐产品、性价比最高产品和不建议购买的类型。

## 项目价值

这个项目把“用户选购需求”转化为“标准检索、证据判断、合规推荐”的自动化流程，适用于消费品选购、企业采购初筛、跨境商品合规调研和垂直导购助手。

相比单纯的商品推荐，它更强调证据链和风险提示，能让用户知道推荐结论从哪里来、哪些信息可靠、哪些地方还需要二次确认。
