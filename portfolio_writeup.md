# 产品标准合规推荐助手：AI Workflow 项目

## 项目一句话介绍

这是一个面向商品选购和采购初筛的 AI Workflow：用户输入产品需求后，系统先查相关标准，再检查商品公开证据，最后生成带证据等级、评分和风险提醒的推荐报告。

## 项目背景

我选择这个项目，是因为很多消费决策并不是“哪个商品更便宜”这么简单。尤其是儿童用品、食品接触材料和企业采购场景，用户真正关心的是：这个商品有没有公开证据证明它符合相关标准。

普通搜索和导购内容通常只告诉用户“这个商品相关”或“销量不错”，但不会系统说明相关标准是什么、商品页面有没有标注执行标准、证据来自哪里、证据可信度有多高。因此我把这个问题设计成一个可运行的 AI Workflow 项目。

## 用户痛点

| 痛点 | 说明 |
|---|---|
| 不知道该查什么标准 | 用户往往只知道产品名称，不知道对应标准体系 |
| 不会判断商品证据 | 商品详情页信息复杂，标准、材质和检测报告不易识别 |
| 推荐理由不透明 | 普通推荐常以价格、销量和口碑为主，缺少合规依据 |
| 大模型容易编造 | 如果不加约束，LLM 可能补全不存在的标准和链接 |

## 我的产品思路

我没有把它做成“问一句，推荐几个商品”的聊天机器人，而是拆成一个有证据约束的工作流：

1. 先把用户需求结构化。
2. 再识别产品品类和合规关注点。
3. 先查标准，再查商品。
4. 对商品公开信息做证据等级判断。
5. 用评分规则排序，而不是让模型凭感觉推荐。
6. 最后输出报告，并保留不确定性和免责声明。

这个设计的核心是降低误推荐风险。对于合规类问题，模型不知道就是不知道，不能靠常识补齐。

## 核心流程

| 步骤 | 作用 |
|---|---|
| 输入校验 | 防止缺少产品名称导致流程无意义 |
| 品类识别 | 判断产品类别、材料和风险点 |
| 标准搜索 | 查国家标准、行业标准和认证信息 |
| 标准整理 | 提取标准编号、适用范围和来源 |
| 商品搜索 | 查品牌官网、电商详情页和检测报告 |
| 合规分析 | 判断商品是否公开标注相关标准 |
| 推荐排序 | 按 100 分制输出推荐等级 |
| 报告生成 | 面向普通用户输出结构化结果 |

## Prompt 设计

Prompt 的重点不是让模型回答得更长，而是让模型回答得更克制。每个 Prompt 都包含：

- Role
- Task
- Input
- Output Schema
- Evidence Rules
- Forbidden Behaviors
- Fallback Rules

我特别强化了三个约束：

1. 标准编号、商品链接、检测报告不能编造。
2. “可能符合”不能写成“确定符合”。
3. 每个商品必须输出 `evidence_level`、`evidence_summary` 和 `uncertainty`。

## 证据链设计

我设计了 A-E 证据等级：

| 等级 | 来源 | 作用 |
|---|---|---|
| A | 政府、国家标准平台、监管机构 | 确认标准真实存在 |
| B | 品牌官网、官方检测报告、官方旗舰店 | 判断商品是否公开标注标准 |
| C | 电商详情页 | 作为商品线索，需要二次确认 |
| D | 第三方文章或问答 | 只能作为补充线索 |
| E | 无公开证据 | 不允许强推荐 |

这个模型让推荐从“模型觉得不错”变成“证据支持到什么程度”。

## 评分规则

评分采用 100 分制：

| 评分项 | 权重 |
|---|---:|
| 标准匹配度 | 40 分 |
| 证据可信度 | 25 分 |
| 用户需求匹配度 | 15 分 |
| 价格合理性 | 10 分 |
| 品牌/渠道可信度 | 10 分 |

我把证据可信度权重提高到 25 分，是因为这个项目的价值不是便宜导购，而是合规初筛。没有证据的商品，即使价格便宜，也不能强推荐。

## 项目价值

这个项目展示了我对 AI 产品的三个理解：

1. AI 工作流要把任务拆细，而不是把所有问题都丢给一个大 Prompt。
2. 涉及合规和购买决策时，证据链比生成能力更重要。
3. 产品经理需要设计异常处理、兜底逻辑和用户可理解的输出边界。

## 当前局限

- 真实搜索质量取决于外部搜索 API。
- 电商详情图中的标准信息可能需要 OCR。
- 标准适用性需要结合产品材质、结构和用途进一步确认。
- 当前版本适合初筛，不替代检测机构或法律意见。

## 后续优化

- 增加标准知识库缓存。
- 接入网页正文抓取和截图留存。
- 增加人工审核节点。
- 输出 PDF/Notion 报告。
- 增加多品类模板，例如儿童餐具、插线板、空气炸锅等。

## 面试讲解版本

我会这样介绍这个项目：

“这是一个面向商品选购和采购初筛的 AI Workflow。我没有直接让大模型推荐商品，而是先让它识别产品品类，再通过搜索节点查相关国家标准、行业标准和认证要求。之后系统会搜索商品页面、品牌官网或检测报告，判断商品有没有公开标注执行标准。最后我设计了证据等级和 100 分评分规则，输出推荐排序和风险提醒。这个项目最核心的价值不是生成一段推荐文案，而是把合规推荐拆成一个有证据链、有异常处理、有风险边界的产品流程。”
